bun-starter is an application to be used as a starting template for other production applications. The pages are intended to illustrate usage. For instance View All Countries illustrates a get api call, Search Countries illustrates a get api call passing a paramter whilst the product pages illustrate POST api calls.
The stack used to build bunstarter is:
  • Bun 1.3.4
  • Lit (to reduce boilerplate for components)
  • SQLite for persistent storage
  • TailwindCSS
  • better-auth library
  • marked library for *.md to *.html
The Bun app is built as a combination static file server and API server using bunserve(). The TailwindCSS colours used have are coded as a custom theme with a primary and secondary pallette along with custom error colours and highlights. These can be viewed from the About page.
Authentication, using the better-auth library, assumes an in house application where the admin user allocates new users and passwords.
A script is included to set up a SQLite database, create the better-auth tables, add a default admin user and populate the test tables for countries and products.
The app also has a file system based, multi category system to show static pages with auto generated index pages. Think blogging system and/or documentation. Pages are displayed 'latest first' and front matter can flag pages as private, draft, read mode and sticky.
There is an upload for images and `*.md` content files.
The pages functionality has a search sytem built in and also has a custom image component to give better control over image wrapping and sizing.
The focus is the creation of desktop applications. Althought the app is responsive, some UI work will look a little off on mobile (send suggested fixes).
The readme.md file contains more details. It will open in a new tab.
